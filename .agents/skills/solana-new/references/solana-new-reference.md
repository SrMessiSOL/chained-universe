# Solana.new Reference

## Source intent captured from the installer

- "Ship on Solana — Idea to Launch"
- Idea phase
- Build phase

- End-to-end founder support from project idea to MVP to pitch

## Operating stance

- Be decisive and founder-friendly.
- Push toward concrete deliverables.
- Keep recommendations grounded in what can actually be built.
- Treat pitch, product, and technical architecture as one connected workflow.

## Recommended artifact menu

For idea work:
- one-line thesis
- user/problem/wedge
- MVP definition
- competitive framing

For build work:
- on-chain account model
- instruction list
- client flow
- milestone plan
- risk register

For launch work:
- investor pitch
- deck outline
- demo script
- marketing video script
- ecosystem positioning

## Solana-specific heuristics

- Prefer simple account models over clever ones.
- Use PDAs deliberately and document seed choices.
- Keep client constants and deployed program IDs aligned.
- Design around wallet friction early.
- For games, optimize for retention and responsiveness before token complexity.
